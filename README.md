# Task Booster

> タスクを無闇に増やすな。
> 1つずつコツコツ取り組もう！

Task Boosterは、作業前に取り組むタスクを選び、作業後に集中度を記録することで、
「やった時間」だけでなく「どれだけ集中できたか」を振り返るためのミニアプリです。

学習や仕事でありがちな、

* やることを増やしすぎる
* 何から始めるか迷う
* 長時間やったのに、手応えが薄い
* “作業した感”だけで満足してしまう

といった状態を避けるために作成しました。

---

## 開発背景

エンジニアデビュー前の学習期間において、
「とにかく準備に時間をかける」だけでは成長に直結しないと感じたことがきっかけです。

作業時間を増やすこと自体は大切ですが、それだけでは
「何に集中したのか」
「集中の質はどうだったのか」
「次に改善すべき点は何か」
が曖昧になりがちです。

そこで、Task Boosterでは以下の流れを重視しました。

1. 今日やるタスクを最大3つに絞る
2. その中から1つを選んで作業を開始する
3. 作業後に集中度を記録する
4. 作業時間と集中度をもとに振り返る

単なるToDoアプリではなく、
「作業量」よりも「集中の質」を意識するための小さなブースターを目指しました。

---

## 主な機能

### タスク登録

今日取り組むタスクを登録できます。
登録できるタスクは最大3件までです。

タスクを増やしすぎると、集中する前に迷子になるため、あえて制限を設けています。

### フォーカスセッション開始

登録したタスクの横にある「開始」ボタンから、作業セッションを開始できます。

開始時刻を記録し、どのタスクに取り組んだかを保存します。

### 作業中画面

現在取り組んでいるタスクと開始時刻を表示します。

「今、自分は何をやっているのか」を明確にするための画面です。

### 集中度記録（実装予定）

作業終了時に、集中度を1〜5で記録できるようにする予定です。

### 自動フィードバック（実装予定）

作業時間と集中度をもとに、簡易的なフィードバックを表示する予定です。

例：

* 短時間かつ集中度が低い場合：タスク分解が甘かった可能性
* 長時間かつ集中度が高い場合：深い集中ができている状態
* 長時間かつ集中度が低い場合：休憩やタスク再設計が必要

---

## 注意事項

現在はH2 Databaseを使用しているため、アプリ再起動時にデータはリセットされます。

そのため、現時点では長期的なデータ保存を目的としたアプリではなく、
タスク選択・集中記録・フィードバックの流れを体験するためのミニアプリとして実装しています。

---

## 工夫した点

### タスクを最大3件に制限

一般的なToDoアプリのように無制限にタスクを登録できるのではなく、
あえて最大3件までに制限しています。

これは、作業前の迷いを減らし、優先順位を明確にするためです。

http://localhost:8080/tasks
